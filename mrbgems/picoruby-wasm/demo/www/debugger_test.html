PicoRuby Debugger Tutorial

Setup

  1. Load the PicoRuby DevTools extension: Chrome menu → Extensions → Load unpacked → mrbgems/picoruby-wasm/debugger/
  2. Open DevTools (F12) and select the PicoRuby tab.
  3. Click the Start Tutorial button below.
  4. Follow the numbered steps in order. Each step tells you exactly what to type in the REPL input at the bottom of the PicoRuby panel.

While paused, the page itself is frozen (WASM is single-threaded), but the DevTools panel runs in a separate extension context and stays responsive. You can freely type commands in the REPL.

Interactive Debugger Tutorial

This tutorial walks you through every debugger feature. The code is split across two files. tutorial_helper.rb defines a method; tutorial_main.rb runs the demo. Together they pause six times at binding.irb.

# tutorial_helper.rb
def double(n)
  binding.irb       # ← Pause 5b: paused inside the method
  n * 2
end

# tutorial_main.rb
# --- Pause 1: basics ---
name     = "PicoRuby"
count    = 42
ratio    = 3.14
langs    = ["Ruby", "C", "JavaScript"]
settings = { debug: true, level: 3 }
binding.irb

# --- Pause 2 & 3: loop iteration ---
results = []
langs.each_with_index do |lang, i|
  upper = lang.upcase
  binding.irb
  results << "#{i}: #{upper}"
end

# --- Pause 5a: step / next ---
binding.irb         # ← paused here (total and answer are nil)
total  = double(count)
answer = "#{name} v#{total}"

answer
Pause 1 — Learning the basics

The prompt changes to irb(debug):NNN> and turns yellow. The right sidebar shows local variables.

1Show help
Type: h
A list of all debug commands and keyboard shortcuts appears.
2Read a variable
Type: name
=> "PicoRuby" — You can evaluate any expression to inspect its value.
3Read another variable
Type: langs
=> ["Ruby", "C", "JavaScript"]
4Call a method
Type: langs.size
=> 3 — Any Ruby expression works, not just variable names.
5Evaluate a complex expression
Type: settings[:level] + count
=> 45 — You can combine multiple variables in one expression.
6Modify a variable
Type: count = 100
=> 100 — The right sidebar updates to show the new value. Changes persist when you resume execution.
7Confirm the change
Type: count
=> 100 — The variable now holds the value you set.
8Continue execution
Type: c
The debugger resumes. The prompt returns to normal irb:NNN> briefly, then immediately pauses again inside the loop (Pause 2).
Pause 2 — First loop iteration

We are now inside the each_with_index block. Check the sidebar — you should see lang, i, upper, and results.

9Inspect loop variables
Type: lang
=> "Ruby" — First iteration.
10Check the index
Type: i
=> 0
11Check the computed value
Type: upper
=> "RUBY"
12Continue to next iteration
Type: c
Execution advances. results gets "0: RUBY" appended, then we pause again at the next iteration (Pause 3).
Pause 3 — Second loop iteration
13Verify the iteration advanced
Type: lang
=> "C" — The second element.
14Check accumulated results
Type: results
=> ["0: RUBY"] — The first iteration's result is already collected.
15Continue past remaining iterations
Type: c
Pauses again inside the third iteration (lang = "JavaScript").
16Continue past the third iteration
Type: c
The loop finishes. We are now at Pause 5a, before total = double(count).
Pause 5a — tutorial_main.rb — before calling double()

We are paused before total = double(count) (in tutorial_main.rb). Both total and answer are not yet assigned.

17Confirm they are not yet computed
Type: total
=> nil — The line has not executed yet.
Type: answer
=> nil — Same reason.
18Check all loop results
Type: results
=> ["0: RUBY", "1: C", "2: JAVASCRIPT"]
19Step INTO double() with s
We are about to execute total = double(count). Type: s
The debugger steps into double and immediately hits the binding.irb inside it — this is Pause 5b.
Notice the pause header now shows tutorial_helper.rb instead of tutorial_main.rb — the debugger crossed a file boundary!
The sidebar shows n = 100 (the argument; you changed count to 100 in Step 6).
Pause 5b — tutorial_helper.rb — inside double()

We are now paused inside double(), about to execute n * 2.

20Inspect the argument, then step out
Type: n to check the argument value.
=> 100
Now type: s
Executes n * 2 and returns to the caller.
The sidebar now shows total = 200, answer = nil.
We are back in tutorial_main.rb, about to execute answer = "#{'{'}name{'}'} v#{'{'}total{'}'}".
21Step OVER the next line with n
We are about to execute answer = "#{'{'}name{'}'} v#{'{'}total{'}'}". Type: n
n (next) executes the entire line without stepping into any method calls inside it.
The sidebar now shows answer = "PicoRuby v200".
Key difference vs s: n would not have entered double() either — but because double contains its own binding.irb, it still would have paused there. For methods without binding.irb, n executes the whole call in one step.
22Finish the tutorial
Type: c
Execution completes. The output box on this page shows the final result. Congratulations — you have learned all the debugger commands!

Tip: You can also use keyboard shortcuts while the REPL panel is focused: F8 = continue, F10 = next, F11 = step.
