# PicoRuby Looper 実装計画

## 目的

`picoruby-midibase`上に、MIDI鍵盤から演奏を録音し、固定長で反復再生できるLooperを実装する。初期音源は`picoruby-psg`とするが、一般的なMIDI音源にも出力できる構造にする。

UIはR2P2上の対話型CUIとし、TUIは将来拡張とする。

## 初期仕様

| 項目         |  デフォルト | 初期対応範囲                 |
| ------------ | ----------: | ---------------------------- |
| テンポ       |     120 BPM | 30〜300 BPM                  |
| 拍子         |         4/4 | 3/4、4/4、6/8、12/8          |
| ループ長     |       4小節 | 1〜16小節                    |
| 時間分解能   |     24 PPQN | 固定                         |
| カウントイン |       1小節 | 0〜4小節                     |
| クォンタイズ |        1/16 | off、1/4、1/8、1/16、1/8三連 |
| トラック声数 |         1声 | PSGでは最大3声               |
| 記録対象     | Note On/Off | velocity、MIDI channelを含む |

拍子、小節数は最初の録音後は変更不可とする。テンポは録音後も変更できる。

## 録音フロー

```text
空のループ:
録音予約 → カウントイン → 指定小節数を録音 → 自動終了 → ループ再生

既存ループあり:
録音予約 → 次のループ先頭まで待機 → 1ループ分録音
          → 自動終了 → 全トラック再生
```

録音終了の合図は不要とする。CUIからの`record`は録音開始ではなく録音予約を意味する。

録音状態でないときもMIDI入力は音源へ送り、演奏音を鳴らす。ただしイベントは記録しない。

## 時間モデル

イベント位置はミリ秒ではなく、ループ先頭からのtickで保持する。

```text
ticks_per_bar = PPQN * 拍子分子 * 4 / 拍子分母
loop_ticks    = ticks_per_bar * 小節数
```

24 PPQNでは四分音符24、八分音符12、十六分音符6、八分三連符8 ticksとなる。MIDI Timing Clockの1クロックとも一致する。

内部クロックを初期実装とし、外部MIDI Clockへの同期は将来対応とする。

## ドンカマ

クリック設定は「回数を2倍」ではなく、次の2段階とする。

| 拍子 |      `beat`           | `subdivision`      |
| ---- | --------------------: | -----------------: |
| 4/4  |   四分音符ごとに4回   | 八分音符ごとに8回  |
| 3/4  |   四分音符ごとに3回   | 八分音符ごとに6回  |
| 6/8  | 付点四分音符ごとに2回 | 八分音符ごとに6回  |
| 12/8 | 付点四分音符ごとに4回 | 八分音符ごとに12回 |

小節頭は異なる音程または音量でアクセントを付ける。

最初の録音ではPSGの1声をクリックに使用するため、録音トラックは最大2声とする。最初のトラック完成後は、デフォルトではクリックを止め、そのトラックを時間の基準として使う。独立したクリック音源は将来検討する。

## トラックとPSGの声

トラックは論理的なMIDI演奏データ、声は音源の物理的資源として分離する。

```text
新規トラックの最大声数 =
  音源の総声数 - 既存トラックに予約済みの声数 - 使用中のクリック声数
```

PSG使用時は、muteされたトラックも予約声数に含める。これにより、unmute時に声が不足する事態を防ぐ。

指定声数を超える同時発音にはlatest-note priorityを適用し、奪われた音符にはNote Offを補って記録結果と再生結果を一致させる。

## 内部イベント形式

受信した生のMIDIバイト列ではなく、解析済みイベントとして保持する。

```text
tick | event type | channel | note | velocity
```

実装ではRubyのHashを並べず、Stringなどに固定長レコードとして格納する。MIDIへの出力時に再びバイト列へ変換する。

初期版はNote On/Offだけを記録する。CC、Pitch Bend、Program Change、SysExはイベント数と実装負荷が大きいため将来対応とする。

## クォンタイズ

Note Onを最寄りのグリッドへ移動し、Note Offは演奏時間を維持するよう同じ量だけ移動する。短い音符が長さ0にならないよう、最小長を1 tickとする。

ループ終端をまたぐ音符は初期版では終端で切る。ループ停止時、トラック削除時、ループ境界の異常時にはAll Notes Off相当の処理を行う。

録音開始よりわずかに早いNote Onを失わないよう、録音予約中はクォンタイズ幅の半分を上限とする先行入力を保持し、ループ先頭へスナップする。

## コンポーネント

```text
MIDI Input → Recorder → Loop / Tracks
                              ↓
Clock → Transport → Player → MIDI Output
                            → PSG Adapter

CUI → Transport / Recorder / Track操作
```

LooperはPSGレジスタを直接操作しない。PSG固有の音程、velocity、音色、声割り当ては`PSG Adapter`が担当する。

MMLは同じTrack/Event形式へコンパイルすることで統合し、Looper本体からは分離する。

## 状態管理

`STOPPED`、`PLAYING`、`ARMED`、`COUNT_IN`、`RECORDING`を持つ。状態遷移時には発音中ノート、現在tick、録音先トラックを明示的に処理する。

## CUI

R2P2から`looper`を起動し、専用の`looper>`プロンプト内で操作する。

主要コマンドは`play`、`stop`、`record [voices]`、`tracks`、`mute N`、`unmute N`、`delete N`、`undo`、`clear`、`tempo N`、`meter N/D`、`bars N`、`quantize VALUE`、`metronome VALUE`、`quit`とする。

演奏とクロック処理を止めずに入力を受けられるよう、LooperエンジンとCUI入力は分離する。

## メモリとエラー処理

各トラックにイベント数上限を設ける。候補は256イベントとし、実測後に決定する。上限超過時は新規録音を破棄し、既存ループを維持したままエラーを表示する。

初期版では既存トラックへの破壊的overdubを行わず、録音ごとに新規トラックを作る。`undo`は直前に追加したトラックを削除する。

## 将来拡張
